**Общая цель индексации**

Оптимизация производительности системы управления продажами и возвратами через создание эффективной структуры индексации данных.

**Конкретные цели индексации по модулям**

**Модуль продаж**

•	Ускорение временных запросов через индексы по sales_time для быстрого анализа продаж по периодам
•	Оптимизация JOIN-операций между таблицами продаж и справочниками через индексы по store_id, item_id, employee_id
•	Повышение эффективности аналитических запросов через составные индексы

**Модуль возвратов**

•	Быстрое получение данных по возвратам через индексацию времени и магазинов
•	Оптимизация поиска по товарам через индексацию item_id
•	
**Справочные данные**

•	Ускорение поиска товаров по категориям и названиям
•	Оптимизация работы с персоналом через индексацию имен сотрудников

**Ожидаемый эффект от индексации**

Производительность:

•	Ускорение типовых запросов в 10-50 раз
•	Снижение нагрузки на CPU при выполнении сложных отчетов
•	Оптимизация использования памяти при работе с большими наборами данных

Практические преимущества:

•	Быстрая фильтрация данных по временным периодам
•	Эффективное объединение таблиц при формировании отчетов
•	Быстрый поиск товаров и сотрудников
•	Оптимизированная работа с возвратами

Анализ влияния на производительность

Позитивные эффекты:
•	Значительное ускорение операций SELECT
•	Оптимизация работы с JOIN
•	Эффективная фильтрация данных
•	Улучшение производительности при группировке

```sql
-- 1. Для фильтрации по дате
CREATE INDEX idx_main_sales_time ON main_sales(sales_time);

-- 2. Для JOIN с магазинами
CREATE INDEX idx_main_sales_store ON main_sales(store_id);

-- 3. Для JOIN с товарами
CREATE INDEX idx_main_sales_item ON main_sales(item_id);

-- 4. Для JOIN с сотрудниками
CREATE INDEX idx_main_sales_employee ON main_sales(employee_id);

-- 5. Составной индекс для часто используемых фильтров
CREATE INDEX idx_main_sales_store_time ON main_sales(store_id, sales_time);

-- ИНДЕКСЫ ДЛЯ ДРУГИХ ТАБЛИЦ ПРОДАЖ

-- com_sales
CREATE INDEX idx_com_sales_time ON com_sales(sales_time);
CREATE INDEX idx_com_sales_store ON com_sales(store_id);
CREATE INDEX idx_com_sales_item ON com_sales(item_id);

-- b2b_sales
CREATE INDEX idx_b2b_sales_time ON b2b_sales(sales_time);
CREATE INDEX idx_b2b_sales_store ON b2b_sales(store_id);

-- ИНДЕКСЫ ДЛЯ ТАБЛИЦ ВОЗВРАТОВ

-- main_returns
CREATE INDEX idx_main_returns_time ON main_returns(return_time);
CREATE INDEX idx_main_returns_store ON main_returns(store_id);
CREATE INDEX idx_main_returns_item ON main_returns(item_id);

-- com_returns
CREATE INDEX idx_com_returns_time ON com_returns(return_time);
CREATE INDEX idx_com_returns_store ON com_returns(store_id);

-- ИНДЕКСЫ ДЛЯ СПРАВОЧНИКОВ

-- Для быстрого поиска товаров по категориям
CREATE INDEX idx_items_category ON items(category_id);

-- Для поиска по названию товара
CREATE INDEX idx_items_name ON items(item_name);

-- Для поиска сотрудников по имени
CREATE INDEX idx_employees_name ON employees(employee_name);
```

После создания индексов проверим их наличие:

```sql
-- Посмотреть все созданные индексы
SELECT 
    name as index_name,
    tbl_name as table_name,
    sql
FROM sqlite_master 
WHERE type = 'index' 
AND name NOT LIKE 'sqlite_%'  -- исключаем системные индексы
ORDER BY tbl_name, name;
```
