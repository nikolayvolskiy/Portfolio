**Проблема:** отсутствие централизованной базы данных, долгое время выполнения ad-hoc задач, отсутствие бюджета на полноценную инфраструктуру.

**Задача:** создать временное решение до развертывания полноценной облачной базы данных, к которому можно будет обращаться для быстрого решения ad hoc задач.

Инструменты: DBeaver

**Ключевые этапы:**

1. Написание скриптов для предобработки данных перед загрузкой в базу.
2. Создание таблиц и ключей.
3. Создание индексов.
4. Заполнение базы.

**Результат:** работающая база, которая сократила время выполнения аналитических ad hoc задач до десятков минут. Денежные затраты - 0.
