## Анализ динамики продаж по точкам продаж (2025 год)

**Цель анализа:**

Проследить, как менялись объёмы продаж в каждом магазине в течение 2025 года, выявить:
-	лидеры и аутсайдеры по выручке;
-	сезонные всплески и спады;
-	тренды роста или падения в отдельных точках.

**Почему это важно:**

-	Позволяет сравнивать эффективность магазинов.
-	Помогает планировать закупки и персонал.
-	Выявляет аномалии (резкие падения/скачки).
-	Даёт основу для принятия решений о развитии сети.

```python
import pandas as pd
import matplotlib.pyplot as plt

# Загрузка данных
df_sales = pd.read_csv('C:/Python/t1_main_sales.csv', sep=';', encoding='cp1251', low_memory=False)
df_stores = pd.read_csv('C:/Python/t1_stores.csv', sep=';', encoding='cp1251', low_memory=False)

#Преобразование даты
df_sales['sales_time'] = pd.to_datetime(df_sales['sales_time'], errors='coerce')

#Фильтрация за 2025 год и добавление месяца
sales_2025 = df_sales[df_sales['sales_time'].dt.year == 2025].copy()
sales_2025['month'] = sales_2025['sales_time'].dt.month

#Добавление названий магазинов
sales_with_stores = sales_2025.merge(df_stores[['store_id', 'store_name']], on='store_id', how='left')

#Группировка по месяцу и магазину
monthly_sales = sales_with_stores.groupby(['month', 'store_name'])['total_value'].sum().reset_index()

#Создание сводной таблицы для графика
pivot_table = monthly_sales.pivot(index='month', columns='store_name', values='total_value')

#Сортировка месяцев
pivot_table = pivot_table.sort_index()

#Построение графика
plt.figure(figsize=(14, 8))

#Создаем линию для каждого магазина
for store in pivot_table.columns:
    plt.plot(pivot_table.index, pivot_table[store], marker='o', linewidth=2, label=store)

plt.xlabel('Месяц (2025 год)', fontsize=12)
plt.ylabel('Сумма продаж, руб.', fontsize=12)
plt.title('Динамика продаж по месяцам 2025 года по точкам продаж', fontsize=14, fontweight='bold')
plt.legend(title='Точки продаж', bbox_to_anchor=(1.05, 1), loc='upper left')
plt.grid(True, alpha=0.3)
plt.xticks(range(1, 13), ['Янв', 'Фев', 'Мар', 'Апр', 'Май', 'Июн', 
                          'Июл', 'Авг', 'Сен', 'Окт', 'Ноя', 'Дек'])

plt.tight_layout()
plt.show()
```

Что делает код:
1.	Загрузка данных
-	df_sales — транзакции продаж;
-	df_stores — информация о магазинах (ID).
2.	Подготовка дат
-	Преобразует столбец sales_time в формат datetime.
-	Фильтрует записи только за 2025 год.
-	Добавляет колонку month (номер месяца).
3.	Связывание продаж с магазинами
-	Объединяет данные о продажах с названиями магазинов по store_id.
4.	Агрегация
-	Группирует по month и store_name, суммирует total_value (выручка).
-	Результат — таблица monthly_sales с колонками:
  -	month — номер месяца;
	- store_name — название магазина;
  -	total_value — сумма продаж за месяц.
5.	Формирование сводной таблицы
o	pivot_table — матрица, где:
-	строки — месяцы (1–12);
-	столбцы — названия магазинов;
-	значения — выручка за месяц.
6.	Визуализация
	Для каждого магазина рисуется отдельная линия.
	Маркеры (o) отмечают точки данных.
	Легенда вынесена вправо для удобства чтения.
	Оси подписаны, сетка добавлена для наглядности.
	Месяцы подписаны сокращёнными названиями (Янв–Дек).
##
Как читать результат
-	Ось X (горизонтальная) — месяцы 2025 года.
-	Ось Y (вертикальная) — сумма продаж в рублях.
-	Каждая линия — динамика продаж одного магазина.
-	Цвет линии — уникальный идентификатор магазина (соответствует легенде).
-	Пересечения линий — моменты, когда один магазин обгоняет другой по выручке.

Примеры интерпретации:
-	Если линия идёт вверх — продажи растут.
-	Если линия плоская — стабильность.
-	Резкие скачки могут указывать на акции или проблемы.
-	Разный наклон линий показывает, какие магазины развиваются быстрее.
