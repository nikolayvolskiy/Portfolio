## Анализ накопительного LTV клиентов по когортам (2024 год)


**Цель анализа:**


Рассчитать накопленный показатель LTV (Lifetime Value) для клиентских когорт 2024 года с помесячной детализацией, выявить:


- динамику роста LTV в разрезе когорт;


- различия в ценности клиентов, привлечённых в разные месяцы;


- периоды максимальной отдачи от клиентов каждой когорты.


**Почему это важно:**


- позволяет оценить долгосрочную ценность клиентов из разных периодов привлечения;


- помогает оптимизировать бюджет на привлечение (определять наиболее выгодные каналы/периоды);


- даёт основу для прогнозирования доходов от текущих и будущих когорт;


- выявляет эффективность удержания клиентов в динамике.


```python

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

#Загрузка данных
df_sales = pd.read_csv('C:/Python/t1_main_sales.csv', sep=';', encoding='cp1251', low_memory=False)

#Преобразование даты
df_sales['sales_time'] = pd.to_datetime(df_sales['sales_time'], errors='coerce')

#Фильтрация за 2024 год
sales_2024 = df_sales[df_sales['sales_time'].dt.year == 2024].copy()

#Создаем колонки с месяцем
sales_2024['month'] = sales_2024['sales_time'].dt.month

#Находим первый месяц покупки для каждого покупателя в 2024 году
first_purchase = sales_2024.groupby('customer_phone')['month'].min().reset_index()
first_purchase.columns = ['customer_phone', 'cohort_month']

#Объединяем с исходными данными
sales_with_cohort = sales_2024.merge(first_purchase, on='customer_phone', how='left')

#Рассчитываем общую выручку от каждого клиента по месяцам
customer_monthly_revenue = sales_with_cohort.groupby(['customer_phone', 'cohort_month', 'month'])['total_value'].sum().reset_index()

#Создаем когорты (группируем по когортному месяцу и месяцу покупки)
cohort_data = customer_monthly_revenue.groupby(['cohort_month', 'month']).agg(
    total_revenue=('total_value', 'sum'),
    n_customers=('customer_phone', 'nunique')
).reset_index()

#Создаем сводную таблицу для выручки
revenue_pivot = cohort_data.pivot_table(
    index='cohort_month',
    columns='month',
    values='total_revenue',
    fill_value=0
)

#Создаем сводную таблицу для количества клиентов
customers_pivot = cohort_data.pivot_table(
    index='cohort_month',
    columns='month',
    values='n_customers',
    fill_value=0
)

#Заполняем отсутствующие месяцы
all_months = range(1, 13)
revenue_pivot = revenue_pivot.reindex(index=all_months, columns=all_months, fill_value=0)
customers_pivot = customers_pivot.reindex(index=all_months, columns=all_months, fill_value=0)

#Вычисляем накопительный LTV
ltv_matrix = pd.DataFrame(index=revenue_pivot.index, columns=revenue_pivot.columns)

for cohort in revenue_pivot.index:
    cohort_size = customers_pivot.loc[cohort, cohort]  #количество клиентов в когорте
    if cohort_size > 0:
        cumulative_revenue = 0
        for month in revenue_pivot.columns:
            if month >= cohort:  #только текущий и последующие месяцы
                cumulative_revenue += revenue_pivot.loc[cohort, month]
                ltv_value = cumulative_revenue / cohort_size
            else:
                ltv_value = 0
            ltv_matrix.loc[cohort, month] = ltv_value
    else:
        for month in revenue_pivot.columns:
            ltv_matrix.loc[cohort, month] = 0

#Преобразуем в float и округляем значения
ltv_matrix = ltv_matrix.astype(float).round(2)

#Преобразуем названия месяцев
month_names = ['Янв', 'Фев', 'Мар', 'Апр', 'Май', 'Июн', 
               'Июл', 'Авг', 'Сен', 'Окт', 'Ноя', 'Дек']

ltv_matrix.index = [month_names[i-1] for i in ltv_matrix.index]
ltv_matrix.columns = [month_names[i-1] for i in ltv_matrix.columns]

#Выводим матрицу LTV
print("Матрица накопительного LTV за 2024 год (в рублях на клиента)")
print("Когорта по вертикали, месяц по горизонтали")
print()
print(ltv_matrix)

#Визуализация - тепловая карта LTV
plt.figure(figsize=(14, 10))

#Определяем максимальное значение для цветовой шкалы
max_ltv = ltv_matrix.max().max()

#Создаем тепловую карту
sns.heatmap(ltv_matrix, 
            annot=True, 
            fmt='.2f', 
            cmap='YlOrRd', 
            vmin=0, 
            vmax=max_ltv,
            linewidths=0.5,
            linecolor='gray',
            cbar_kws={'label': 'LTV (руб.)'})

plt.title('Матрица накопительного LTV по месяцам 2024 года', 
          fontsize=16, fontweight='bold', pad=20)
plt.xlabel('Месяц', fontsize=12)
plt.ylabel('Когорта (месяц привлечения)', fontsize=12)
plt.xticks(rotation=0)
plt.yticks(rotation=0)
plt.tight_layout()
plt.show()

```

**Что делает код:**

Загрузка данных:

df_sales — транзакции продаж с информацией о клиентах.

Подготовка дат:

 - преобразует столбец sales_time в формат datetime;


- фильтрует записи только за 2024 год;


- добавляет колонку month (номер месяца).


Формирование когорт:


- определяет первый месяц покупки (cohort_month) для каждого клиента;


- объединяет данные о продажах с информацией о когортах.


Агрегация выручки:


- группирует данные по клиенту, когорте и месяцу, суммирует total_value;


Результат — таблица customer_monthly_revenue с колонками:


- customer_phone — идентификатор клиента;


- cohort_month — месяц первой покупки;


- month — месяц транзакции;


- total_value — сумма покупок за месяц.


Создание сводных таблиц:


- revenue_pivot — матрица выручки по когортам и месяцам;


- customers_pivot — матрица количества клиентов в когортах по месяцам.


Расчёт накопительного LTV:


- для каждой когорты накапливает выручку от месяца привлечения и далее;


- делит накопленную выручку на размер когорты (cohort_size);


- заполняет нулями значения для месяцев до привлечения когорты.


Форматирование:


- округляет значения LTV до 2 знаков после запятой;


- заменяет номера месяцев на сокращённые названия (Янв–Дек).


Визуализация:


- строит тепловую карту (heatmap) с цветовой шкалой от жёлтого к красному;


- отображает числовые значения LTV в ячейках;


- подписывает оси и добавляет заголовок.


**Как читать результат:**


- Строки матрицы — когорты (месяцы привлечения клиентов).


- Столбцы матрицы — месяцы анализа.


- Цвет ячейки — величина LTV (чем краснее, тем выше значение).


- Число в ячейке — накопленный LTV в рублях на одного клиента когорты к данному месяцу.


- Диагональные значения — LTV на момент месяца привлечения (обычно минимальны).


- Правые столбцы для каждой строки — итоговый LTV когорты за 2024 год.


**Примеры интерпретации:**


Если значения в строке растут — клиенты из данной когорты продолжают совершать покупки, их накопленная ценность увеличивается со временем. Чем круче подъём, тем активнее повторные продажи.


Если строка выходит на плато — рост LTV остановился, клиенты перестали приносить дополнительную выручку. Это может означать:


- завершение жизненного цикла клиентов когорты;


- отсутствие стимулов для повторных покупок;


- потерю интереса к продукту.


Если значения в столбце увеличиваются сверху вниз — когорты, привлечённые в более поздние месяцы, показывают более высокий LTV к этому сроку. Это может быть связано с:


- улучшением качества привлечения (более целевые клиенты);


- изменениями в продукте/сервисе, повышающими удержание;


- сезонными факторами (например, предновогодние покупатели активнее тратят).


Если значения в столбце уменьшаются сверху вниз — ранние когорты ценнее поздних. Возможные причины:


- снижение качества трафика;


- ухудшение клиентского опыта;


Ярко-красные ячейки в правой части матрицы — когорта демонстрирует высокий долгосрочный потенциал. Такие клиенты приносят значительную выручку спустя месяцы после первой покупки.


Бледные ячейки даже в конце периода — низкая отдача от когорты. Стоит проанализировать:


- каналы привлечения этих клиентов;


- их средний чек и частоту покупок;


- причины оттока.


Диагональный градиент (от бледного к яркому) — типичный паттерн: LTV растёт по мере увеличения возраста когорты. Чем дольше клиент с компанией, тем выше его ценность.


Резкие скачки в строке — возможны акции, сезонные всплески или единичные крупные покупки, искажающие картину. Рекомендуется проверить аномалии на предмет:


- ошибок в данных;


- исключительных событий (например, корпоративный заказ);


- эффективности временных маркетинговых активностей.


Низкий LTV для всех когорт — системный сигнал:


- цены не соответствуют ценности продукта;


- слабая программа лояльности;


- высокая конкуренция, заставляющая клиентов искать альтернативы.


Ключевые выводы из тепловой карты:


- Самые ценные когорты — те, где ячейки в правой части матрицы максимально насыщены цветом.


- Проблемы удержания видны по строкам, которые рано выходят на плато или имеют разрывы.


- Сезонные эффекты проявляются в столбцах с аномально высокими/низкими значениями для всех когорт.


- Эффективность изменений в продукте/маркетинге можно отследить по улучшению LTV в поздних когортах.
